Common questions

Everything you need to know
about Conversion Optimisation.

What is conversion rate optimisation (CRO)?
Conversion rate optimisation (CRO) is the systematic process of increasing the percentage of website visitors who take a desired action — filling out a form, booking a demo, making a purchase, or calling your business. It involves analysing user behaviour, identifying friction points in your funnel, running A/B tests, and implementing the variations that produce measurably better results. AI-powered CRO automates much of this process so testing runs continuously rather than in occasional manual sprints.
How long does an A/B test need to run to be valid?
A statistically valid A/B test typically requires a minimum of 1–2 weeks and enough traffic to achieve 95% statistical significance — usually at least 500–1,000 conversions split across variants. Our system monitors tests in real time and automatically flags when significance thresholds are met, so you never call a winner too early or run a test longer than necessary.
What elements of a website can be tested?
Almost anything that affects user behaviour can be tested — headlines, hero copy, CTA button text and placement, form length and fields, page layout, pricing presentation, social proof placement, imagery, and page load speed. Our AI identifies which elements are most likely to impact conversion based on heatmap data, session recordings, and behavioural analytics before prioritising what to test first.
How quickly can we expect to see conversion improvements?
Most clients see their first statistically significant test result within 4–6 weeks. Quick-win optimisations — fixing obvious friction points like slow forms, unclear CTAs, or mobile layout issues — often deliver improvements within the first 2 weeks. The compounding effect of continuous testing typically delivers a 2.3× conversion lift within 6 months.
Do we need high website traffic for CRO to work?
You need enough traffic to generate statistically meaningful results, but you don't need massive volume. Sites with 1,000+ monthly visitors can run meaningful tests on high-traffic pages. For lower-traffic sites, we focus on qualitative analysis — heatmaps, session recordings, user interviews — to identify and fix high-confidence issues before moving into A/B testing as traffic grows.
